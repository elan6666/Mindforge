# Plan 10-02: Reviewer Loop, Frontend Support, and Ark Smoke Test

## Objective

Implement the end-to-end `paper-revision` loop, expose it in the web app, and verify the OpenAI-compatible Ark/Doubao adapter path.

## Tasks

- Add a paper-specific serial stage flow: standards analysis, revision draft, style review, content review, revision iteration, and final re-review.
- Update the `paper-revision` preset and default rule template to use explicit paper roles.
- Register Volces Ark and `doubao-seed-2.0-lite` in the model catalog.
- Add `model-api` adapter mode for OpenAI-compatible `/chat/completions`.
- Add frontend journal/reference-paper inputs and an Academic Context result tab.
- Add backend, adapter, frontend, and build tests.

## Verification

- Backend tests verify the paper revision loop and model routing.
- Frontend tests verify academic context appears in task details.
- A live Ark smoke test verifies `doubao-seed-2.0-lite` can be called without storing secrets in the repository.
