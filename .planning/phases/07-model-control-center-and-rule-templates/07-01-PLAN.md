# Plan 07-01

## Objective

实现后端模型中心可编辑配置层、规则模板 schema 与 coordinator-driven assignment 基础服务。

## Tasks

1. 设计“基础 catalog + 本地 override/rule config”的配置结构和 schema
2. 增加模型中心相关 API，至少覆盖：
   - 查询当前可编辑模型状态
   - 更新 priority / enabled
   - 查询规则模板
   - 创建/更新/删除规则模板
3. 新增 coordinator selection service，用于根据任务、preset、task type 和模板候选解析本次命中的规则模板
4. 扩展任务请求/响应元数据，使执行结果能记录：
   - resolved rule template
   - coordinator model
   - role assignments
5. 让 `TaskService` 和多阶段编排先支持“消费规则模板结果”，即使初始版本仍保留显式 override 优先

## Key Files (expected)

- `app/backend/schemas/rule_template.py`
- `app/backend/services/model_control_service.py`
- `app/backend/services/rule_template_service.py`
- `app/backend/services/coordinator_selection_service.py`
- `app/backend/api/routes/model_control.py`
- `app/backend/schemas/task.py`
- `app/backend/services/task_service.py`

## Verification Focus

- 后端可查询并更新模型优先级和启停状态
- 后端可创建并返回结构化规则模板
- 任务执行结果能返回命中的 rule template 和分配结果
