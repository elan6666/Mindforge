# Plan 08-02

## Objective

实现任务/阶段日志持久化和前端历史查看。

## Tasks

1. 设计 task_run 和 stage_run 的 SQLite schema
2. 把当前执行结果和阶段轨迹写入持久化层
3. 增加历史查询 API，至少支持：
   - 最近任务列表
   - 按状态筛选
   - 单任务详情
4. 在前端工作台中增加历史/详情视图
5. 在详情里展示 task metadata、stage summaries 和 approval record

## Key Files (expected)

- `app/backend/schemas/history.py`
- `app/backend/services/history_service.py`
- `app/backend/api/routes/history.py`
- `frontend/src/App.tsx`
- `frontend/src/lib/api.ts`
- `frontend/src/types.ts`

## Verification Focus

- 任务和阶段日志能落库并重新读出
- 前端能看到最近任务列表
- 详情视图能显示阶段、模板、模型和审批信息
