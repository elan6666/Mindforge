# Plan 06-02

## Objective

实现聊天式任务区和右侧结果面板，把当前后端 API 接进可用 UI。

## Tasks

1. 实现聊天主区和任务输入组件
2. 增加常见控件：preset 选择、repo_path、默认协调模型、提交按钮
3. 接入 `/api/tasks` 与 `/api/presets`
4. 接入 `/api/providers` 与 `/api/models`，至少用于只读选择或展示
5. 实现右侧结果/标签面板：final output、orchestration trace、repo summary、task metadata
6. 为执行中、失败、空状态提供基本反馈

## Key Files (expected)

- `frontend/src/components/chat/*`
- `frontend/src/components/task-controls/*`
- `frontend/src/components/panels/*`
- `frontend/src/lib/api/*`
- `frontend/src/state/*`

## Verification Focus

- 可以通过 UI 提交任务
- 可以看到结果和阶段轨迹
- 常见状态反馈清晰，不需要打开开发者工具才能理解
